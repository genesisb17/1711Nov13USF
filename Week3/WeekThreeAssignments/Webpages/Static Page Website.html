What is SQL?

In order to understand what is SQL, we must first give definition to what a database is. According to the Merriam-Webster dictionary, a database is a usually large collection of data organized especially for rapid search and retrieval.


A database is composed of tables with many fields. These fields are basically pieces of data that when put together can create a plethora of information regarding an specific entry in the table.


The way I like seeing it, we're all walking databases. We all have a lot of information that we store in our minds. We can access that information and communicate it to others in many ways (talking, drawing it, writing it down, etc). In the same way we, as walking databases, can communicate the information we have stored in our minds to others, so can databases!


This is where SQL comes into play!


Structured Query Language, also known as SQL, is the language we use to communicate with databases. We can refer to SQL as the language we use to speak with databases. This language we use to communicate with databases consists of 5 sublanguages that focus on different forms of communication.


Data Control Language

Data Control Language, also referred to as DCL, is an SQL sublanguage that is used to control who is provided access to the database.

There are 2 commands relevant to this sublanguage.

  • GRANT
  • REVOKE

GRANT


The GRANT command gives permission to an specific user for access to an specific database. The syntax for the command is listed below.


GRANT privilege_name ON object_name

TO {user_name | PUBLIC | role_name} [with GRANT option]


  1. privilege_name
    • is the access right or privilege granted to the user.
  2. object_name
    • is the name of the database object like table, view, etc.
  3. user_name
    • is the name of the user to whom an access right is being granted.
  4. With Grant option
    • allows users to grant access rights to other users.

Public is used to grant all database rights to the users

REVOKE


The REVOKE command is used to remove user access rights or privileges to the database objects. The syntax for the command is listed below.


REVOKE privilege_name ON object_name

FROM {User_name | PUBLIC | Role_name}


  1. privilege_name
    • is the access right or privilege granted to the user.
  2. object_name
    • is the name of the database object like table, view, etc.
  3. user_name
    • is the name of the user to whom an access right is being granted.

Data Manipulation Language

Data Manipulation Language, also referred to as DML, is an SQL sublanguage that is used to manipulate and access the database.

There are many commands relevant to this sublanguage. However, we will only be explaining the most common ones.

  1. SELECT
  2. UPDATE
  3. INSERT
  4. DELETE

SELECT


The SELECT command is the basis of the DML language. It indicates what columns from an specific table you will be returning. The syntax for a simple command is listed below.


SELECT column_name FROM table_name WHERE condition


  1. column_name
    • refers to the name of the column to be returned from table.
  2. table_name
    • refers to the name of the table which the column will be chosen from
  3. condition
    • refers to an specific condition that needs to be met for rows to be returned from table

UPDATE


The UPDATE command lets you control what updates are to be made to a record in a table. It does not return any rows of information, but if you're using an IDE you should receive a message indicating how many rows were update. The syntax for a simple UPDATE command is listed below.


UPDATE table_name

SET column_name1 = value1, column_name2 = value2,...

WHERE condition


  1. table_name
    • refers to the name of the table which the column will be chosen from
  2. column_nameX, valueX
    • refers to the column and specific value that will be given to the row that meets given condition.
  3. condition
    • refers to an specific condition that determines which row will be updated.

INSERT


The INSERT command lets you insert a new entry into the table on a database. It does not return any rows of information, but if you're using an IDE you should receive a message indicating how many rows were inserted. The syntax for a simple INSERT command is listed below.


INSERT INTO table_name(column1,column2,column3,...)

VALUES (value1, value2,value3,...


  1. table_name
    • refers to the name of the table which the new values will be inserted to.
  2. columnX, valueX
    • refers to the column and specific value that will be given to that specific column on the new entry

DELETE


The DELETE command lets you delete existing records of a table in a database. It does not return any rows of information, but if you're using an IDE you should receive a message indicating how many rows were deleted. The syntax for a simple DELETE command is listed below.


DELETE FROM table_name

WHERE condition


  1. table_name
    • refers to the name of the table which the delete command will be taking place in.
  2. condition
    • refers to the condition of the specific entry/ies that will be deleted

Data Definition Language

Data Definition Language, also referred to as DDL, is an SQL sublanguage that is used to create the schema of your database.

There are many commands relevant to this sublanguage, but we will be covering 4

  1. TRUNCATE
  2. ALTER
  3. CREATE
  4. DROP

TRUNCATE


The TRUNCATE command deletes all of the data within the table, but not the table itself. The syntax for executing a TRUNCATE command is listed below


TRUNCATE TABLE table_name


  1. table_name
    • refers to the table where all the data will be deleted.

ALTER


The ALTER command changes the layout of an specific table by either adding, removing, or modifying a column or even adding or removing constraints on an existing table. The syntax for executing an ALTER command is listed below


ADD

ALTER TABLE table_name

ADD column_name datatype


  1. table_name
    • refers to the table that will be altered.
  2. column_name datatype
    • refers to the name of the new column that will be inserted and the datatype that column will have.

DROP

ALTER TABLE table_name

DROP column_name


  1. table_name
    • refers to the table that will be altered.
  2. column_name
    • refers to the name of the column that will be removed.

ALTER

ALTER TABLE table_name

ALTER COLUMN column_name datatype


  1. table_name
    • refers to the table that will be altered.
  2. column_name datatype
    • refers to the name of the column who's datatype will be changed to the given datatype.

CREATE


The CREATE command creates a new database or a new instance of a table in a database. The syntax for executing a CREATE command is listed below


CREATE database

CREATE database database_name


  1. database_name
    • refers to the name of the database that will be created

CREATE TABLE

CREATE TABLE table_name(

column_name1 datatype1, column_name2 datatype2, column_name3 datatype3, ... );


  1. table_name
    • refers to the name of the table that will be created
  2. column_nameX datatypeX
    • refers to the name of the column that will be created and the datatype the column will have.


DROP


The DROP command deletes all of the data within the table AND the table itself. The syntax for executing a DROP command is listed below


DROP TABLE table_name


  1. table_name
    • refers to the table that will be removed from the database.

Be careful before dropping a table. Deleting a table will result in loss of complete information stored in the table!

Data Query Language

Data Query Language, also referred to as DQL, is the commands that are used to retrieve data from the database.

The only command relevant to this sublanguage is listed below


SELECT


The SELECT command is the basis of the DML language. It indicates what columns from an specific table you will be returning. The syntax for a simple command is listed below.


SELECT column_name FROM table_name WHERE condition


  1. column_name
    • refers to the name of the column to be returned from table.
  2. table_name
    • refers to the name of the table which the column will be chosen from
  3. condition
    • refers to an specific condition that needs to be met for rows to be returned from table

Transaction Control Language

Transaction Control Language, also referred to as TCL, refers to the commands that are used to control the transactions made against the database.

The 3 commands relevant to this sublanguage are listed below

  1. COMMIT
  2. ROLLBACK
  3. SAVEPOINT

COMMIT


The COMMIT command brings an end to your current transaction and makes permanent all the changes performed in the transaction. The syntax for a commit command is listed below.


COMMIT;


ROLLBACK


The ROLLBACK command undoes transacations that haven't been committed to the database. This command can only be used to undo transactions since the last commit or rollback that was called. The syntax for a rollback command is listed below.


ROLLBACK;


SAVEPOINT


The SAVEPOINT command creates a point in a transaction you can rollback to without rolling back the entire transaction. In other words, a checkpoint. The syntax for a savepoint creation is listed below.


SAVEPOINT savepoint_name;

  1. savepoint_name
    • refers to the name of the savepoint that will be created.
ROLLBACK TO SAVEPOINT

ROLLBACK TO savepoint_name

  1. savepoint_name
    • refers to the name of the savepoint that will be created.
