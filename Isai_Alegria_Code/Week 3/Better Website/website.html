Introduction to Object Oriented Programming




  • Relies on procedure calls (routines or subroutines) that focus on order and execute in linear fashion, otherwise referred to as "top-down" languages.
  • Hard to reuse functions because the function you want to copy probably references headers, global variables and other other functions.
  • Not suitable for high-level abstraction since mainly low level structures.


  • Focuses on components that the user perceives with objects are basic unit.
  • Easier to design software since you are dealing with higher-levels of concepts and can focus on the task at hand rather than the machine's bytes.
  • We can easily debug programs since they are easier to understand
  • Code easier to reuse.


Abstraction



  • Practice of only showing what is relatable to the user. All trivial and/or non-essential units aren't available to user.
  • Also identifying the main properties so we can better distinguish two objects.
  • Achieved by implementing interfaces and abstract classes.

  • An interface can extend another Java interface only, an abstract class can extend another Java class and implement multiple Java interfaces.
  • Abstract class can provide the implementation of interface. Interface can’t provide the implementation of abstract class.
  • Interface can have only abstract methods. Abstract class can have abstract and non-abstract methods


Encapsulation




  • Encapsulation in Java is a mechanism of wrapping the data and methods together as a single unit
  • Also known as data hiding, protect information from being unintetionally manipulated.
  • Provide public setter and getter methods to modify and view the variables values.

  • Encapsulation and abstraction go hand in hand so much so that you could make the point that they are truly inseparable.
  • Abstraction is the "abstract form of anything", so this is the process of generalzing.
  • Encapsulation more focused on how the inside works but the goal is to hide the low-level implementation.


Inheritance


"inherit" definition: A quality, characteristic, or other immaterial possession, received from predecessors by succession.



  • A subclass will "inherit" the methods and fields of the superclass that it extends.
  • extends is the keyword used to inherit the properties of a class.

  • Inheritance represents the IS-A relationship, also known as parent-child relationship.
  • There are different types of inheritance. From directly from the superclass is called "single" level inheritance. When a class inherits from a class that inherits from a class is "multi-level" inheritance. Finally, we have hierarchical inheritance, where two classes inherit from a single superclass.
  • We use inheritance through method overriding.


Polymorphism




  • Java’s ability to process objects of various types and classes through a single, uniform interface.
  • Compile time polymorphism (static binding) and Runtime polymorphism (dynamic binding). Method overloading is an example of static polymorphism, while method overriding is an example of dynamic polymorphism.
  • An important example of polymorphism is how a parent class refers to a child class object. In fact, any object that satisfies more than one IS-A relationship is polymorphic in nature.



  • Inhertance defines parent-child relationships, while polymorphim takes advantage of that relationship to provide more dynamic code.
  • Inheritance is meant for inital methods to be reused in subclasses, polymorphism allows a child to redefine already defined behavior.
  • Java doesn't allow multiple inheritance of classes, but allows multiple inheritance of Interface, which is required to implement polymorphim.
