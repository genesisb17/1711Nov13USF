Strings are pretty important in Java. They are not a primitive data type but rather objects of the String class. They have some unique properties.

  • Strings are immutable. This means that once a string object is created, it cannot be changed. If the variable it is assigned to becomes assigned to another string object, the original string object with no reference in the String pool is ready for garbage collection.
  • String variables cannot be directly compared. Due to strings being objects, string variables are only references pointing to objects. If one were to use = to determine if two strings held the same value, the operator would instead check if the references were the same. This can be the case sometimes due to the way the string pool operates but most often it is not. A way to handle whether two Strings are equal is covered on the Popular Methods page.
  • Strings can be concatenated with the + operator. This can't be done with other objects instantiated from other classes. The operator is overloaded to specifically work for Strings.
